## 1. Strategic Context 🌍

This operation governs the evaluation, integration, and mastery of the advanced AI agent ecosystem. This plan formalizes a **Three-Tiered Doctrine** for AI resource management, ensuring the right tool is used for the right job, maximizing capability while minimizing cost.

---

## 2. Phase 1: Foundation & Unification 🏗️

### Strategic Imperative:
Establish a single, resilient command layer for all AI interactions and implement our tiered, multi-provider strategy.

- [ ] **Task ID: BF-P1-T1 - Deploy Unified Agent Controller (Terminal Jarvis)**
  - **Description:** Implement Terminal Jarvis as the single entry point for all developer-initiated AI agent tasks.
  - **Done means Done:** The `jarvis` CLI is installed and configured. It can successfully detect and launch our primary AI tools (`llxprt-code`, `qwen-code`, and any CLI provided by Copilot).

- [ ] **Task ID: BF-P1-T2 - Implement Three-Tiered Agent Doctrine**
  - **Description:** Configure our suite of AI tools to reflect the official Three-Tiered Doctrine for resource management.
  - **Done means Done:**
    - **Tier 1 (Free Workhorses):** `LLxprt Code` is configured with profiles for our **OpenRouter key** (defaulting to a free model) and a dedicated profile for the free **Qwen Coder**.
    - **Tier 2 (Integrated Co-Pilot):** The **GitHub Copilot** extension in VS Code is confirmed as the primary tool for all in-IDE, developer-velocity tasks.
    - **Tier 3 (Pro-Tier Heavy Artillery):** `LLxprt Code` has a dedicated, clearly marked profile for the **Gemini Pro API key**, to be used only for specialized, high-value tasks.
    - All profiles are tested to verify connectivity.

---

## 3. Phase 2: Workflow Integration & Mastery 🚀

### Strategic Imperative:
Adopt and standardize advanced agentic workflow frameworks to increase the sophistication and reliability of our AI-driven development.

- [ ] **Task ID: BF-P2-T1 - Formalize Lifecycle Commands (SuperClaude Framework)**
  - **Description:** Integrate the SuperClaude command structure (`/build`, `/test`, `/review`, `/deploy`) into our official Bifrost Protocol.
  - **Done means Done:** The Bifrost Protocol document is updated to mandate that all task specifications for AI agents must be framed using this command structure.

- [ ] **Task ID: BF-P2-T2 - Implement Context-Aware Analysis (iFlow CLI)**
  - **Description:** Evaluate and integrate `iFlow CLI` for its repository analysis and context interpretation capabilities.
  - **Done means Done:** A new step is added to the Vulcan Protocol for "Contextual Sanity Check," where `iFlow CLI` is used to analyze the codebase before a complex task is assigned to an implementation agent.
