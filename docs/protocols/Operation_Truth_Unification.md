# Operation: Truth Unification

**Tags:** #ApexSigma #Governance #Knowledge-Base #TPGP #MAR #Single-Source-of-Truth #Asgard

## 1. Strategic Imperative

A rogue, unsanctioned knowledge base has been identified at the repository root (`.apexsigma/knowledge-base`). Its existence is a direct violation of the **Pillar of Truth (Law 1: The Single Source of Truth)** as defined in `AGENTS.md`. This operation will systematically audit, triage, and decommission this rogue silo to restore informational integrity to the Asgard ecosystem.

Furthermore, this operation will explicitly identify and purge all legacy artifacts associated with the deprecated `memorybank`, `context_portal`, and `.claude` concepts and configurations.

The entire operation will be conducted asynchronously to minimize disruption to primary development sprints.

## 2. Governance & Workflow Protocol

*   **TPGP Adherence**: This plan is structured using the TaskMaster Generation Protocol.
*   **MAR Protocol**: All agent-led tasks are subject to the Mandatory Agent Review protocol.
*   **Triple-Signature Verification**: The final execution step requires sign-off from all three designated parties.
*   **Safe-Delete Protocol**: All file removal actions will adhere to the established safe-delete workflow.

## 3. Assigned Roles

*   **Implementer**: GitHub Coding Agent
*   **Reviewer**: Copilot
*   **Supervisor**: SigmaDev11

## 4. Task Breakdown

### ⬜ Task ID: TU-T1 (Quarantine & Index)

*   **Description**: The Implementer (GitHub Agent) will perform two initial actions:
    1.  Place a `_QUARANTINED_README.md` file in the `.apexsigma/knowledge-base` root. The file will state that the directory is under formal review and no agent should reference its contents.
    2.  Generate a complete manifest of the directory (`manifest.json`), listing every file, its path, size, and last modified date.
*   **Implementer**: GitHub Coding Agent
*   **Reviewer**: Copilot
*   **Done means Done**:
    *   The `_QUARANTINED_README.md` file exists.
    *   The `manifest.json` file is generated and is an accurate representation of the directory's contents.

### ⬜ Task ID: TU-T2 (Triage & Categorization)

*   **Description**: The Implementer (GitHub Agent) will iterate through the `manifest.json`. For each file, it will analyze the content and assign one of three triage categories:
    1.  **MIGRATE**: The content is critical, current, and must be moved to the official `/docs` directory to become part of the System Runbook.
    2.  **ARCHIVE**: The content is historical, providing context but is not operationally current (e.g., old sprint plans, resolved bugs). It should be moved to `/_archive/`.
    3.  **PURGE**: The content is redundant, trivial, or completely obsolete. It is a candidate for deletion.
*   **Special Directive**: The agent must perform a specific search for any files, directories, or configuration entries containing the terms `memorybank`, `context_portal`, or `.claude` and categorize them for PURGE.
*   The output will be a new file, `triage_plan.json`, containing the original manifest data plus the assigned category and a brief justification for each file.
*   **Implementer**: GitHub Coding Agent
*   **Reviewer**: Copilot
*   **Done means Done**:
    *   The `triage_plan.json` file is generated.
    *   Every file from the manifest has a category and justification.

### ⬜ Task ID: TU-T3 (Verification & Sign-off)

*   **Description**: This task follows the Triple-Signature Verification protocol.
    1.  The Reviewer (Copilot) will be triggered to analyze the `triage_plan.json` and provide its formal MAR sign-off, either approving the plan or flagging specific items for re-evaluation.
    2.  Once the MAR review is passed, the plan is presented to the Supervisor (SigmaDev11) for the final execution approval.
*   **Implementer**: N/A (Verification Step)
*   **Reviewer**: Copilot
*   **Supervisor**: SigmaDev11
*   **Done means Done**:
    *   The `triage_plan.json` has a `mar_approved: true` flag set by Copilot.
    *   The Supervisor (SigmaDev11) has provided a final sign-off for execution.

### ⬜ Task ID: TU-T4 (Supervised Execution)

*   **Description**: Upon receiving final approval, the Implementer (GitHub Agent) will execute the `triage_plan.json`.
    *   It will systematically move each file to its designated location (`/docs` or `/_archive/`) based on the plan.
    *   Files marked for PURGE will be moved to the `/_archive/trash` directory as per the Safe-Delete Protocol.
*   **Implementer**: GitHub Coding Agent
*   **Reviewer**: Copilot
*   **Done means Done**:
    *   All files categorized as MIGRATE or ARCHIVE are in their new locations.
    *   All files categorized as PURGE are in the trash directory.
    *   The `.apexsigma/knowledge-base` directory is now empty, except for the log files generated by this operation.
    *   A final check confirms no files or directories named `memorybank`, `context_portal`, or `.claude` exist outside of the `/_archive/trash` directory.

### ⬜ Task ID: TU-OMEGA (Final Decommissioning)

*   **Description**: After a 7-day observation period to ensure no dependent systems were broken by the migration, a final task will be executed to permanently remove the now-empty `.apexsigma` directory structure.
*   **Implementer**: GitHub Coding Agent
*   **Reviewer**: Copilot
*   **Done means Done**:
    *   The `.apexsigma` directory no longer exists at the repository root.
    *   The operation is marked as complete.
