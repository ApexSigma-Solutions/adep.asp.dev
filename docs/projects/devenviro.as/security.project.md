Security Policy for DevEnviro
We take the security of our project seriously. If you discover a security vulnerability, we appreciate your help in disclosing it to us privately so we can address it before it becomes public knowledge.

Supported Versions
Since DevEnviro is under active development, only the latest version is supported. Security updates will be applied to the most recent commit on the main branch.

Version	Supported
latest	:white_check_mark:

Reporting a Vulnerability 🛡️
Please do not report security vulnerabilities through public GitHub issues.

Instead, please send an email to **security@devenviro.example.com**  <!-- update to real mailbox -->
Please provide a detailed report with the following information if possible:

A description of the vulnerability and its potential impact.

Steps to reproduce the issue, including any scripts, code snippets, or configurations.

The specific version or commit hash where the vulnerability was discovered.

Any potential mitigations or workarounds you've identified.

Our Commitment
Once a vulnerability is reported, we are committed to the following process:

We will acknowledge receipt of your report within 48 hours.

We will investigate the issue and confirm the vulnerability.

We will work on a patch and provide you with an estimated timeline for the fix.

We will notify you when the patch has been released. We may also issue a public disclosure and credit you for the discovery if you agree.

Scope
This security policy applies to the code within this project repository and the official Docker images built from it. It does not apply to the security of third-party services used by the project, such as external LLM APIs or the base Docker images for services like RabbitMQ and PostgreSQL.
